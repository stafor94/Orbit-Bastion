# Orbit Bastion

모바일 우선 브라우저 타워디펜스 프로토타입입니다.

## 실행

배포된 GitHub Pages 주소에서 바로 접속할 수 있습니다.

```text
https://stafor94.github.io/Orbit-Bastion/
```

브라우저에서 `index.html` 파일을 직접 열어도 됩니다. 별도의 Python 서버나 Pyodide CDN 로딩 없이 JavaScript만으로 게임이 실행됩니다.

```text
file:///.../Orbit-Bastion/index.html
```

일반 정적 서버에서도 실행할 수 있지만 필수는 아닙니다.

## 배포와 캐시

`index.html`은 CSS와 JavaScript를 불러올 때 고정 자산 버전(`2026-06-content-sync`)을 쿼리 문자열로 붙입니다. 배포 후 파일이 바뀌면 이 값을 함께 변경해 브라우저 캐시를 갱신하세요.

개발 중 강제로 새 파일을 받거나 특정 배포본을 확인하려면 URL에 `?v=원하는값`을 붙일 수 있습니다.

```text
file:///.../Orbit-Bastion/index.html?v=dev
https://stafor94.github.io/Orbit-Bastion/?v=2026-06-content-sync
```

## 검증

수치 계산과 콘텐츠 데이터 회귀를 한 번에 확인하려면 아래 명령을 실행합니다.

```bash
node scripts/run_tests.js
```

개별 검증도 실행할 수 있습니다.

```bash
node scripts/towerMetrics.test.js
node scripts/gameData.validation.test.js
node scripts/sync_readme_content.js --check
```

`gameData.validation.test.js`는 스테이지와 규칙 수 동기화, 스테이지 좌표 범위, 적/타워/연구 참조 무결성, 난이도/타워/분기 구조를 확인합니다.

콘텐츠 수량과 이름이 바뀌면 아래 명령으로 `구현됨` 목록을 데이터 파일 기준으로 자동 갱신할 수 있습니다.

```bash
node scripts/sync_readme_content.js
```

## 2차 작업 내용

- 클리어한 스테이지와 잠긴 스테이지를 구분하는 기지 화면을 추가했습니다.
- 연구 점수로 코어 내구도, 시작 합금, 타워별 성능을 강화할 수 있습니다.
- 연구 효과는 전투 시작 능력치와 타워 피해/범위/감속 계산에 반영됩니다.
- 전투 종료 시 처치 수, 획득 합금, 연구 보상을 보여주는 결과 화면을 추가했습니다.

## 3차 작업 내용

- 보스 전용 체력바와 중앙 경고 연출을 추가했습니다.
- 브루드 콜로서스는 체력 70%, 40%, 15% 구간에서 페이즈가 상승하고 소환 압박이 강해집니다.
- 타워 선택 패널에 피해, 사거리, 쿨타임, 폭발 범위, 감속률 같은 실제 전투 수치를 표시합니다.
- 판매 버튼은 실수 방지를 위해 한 번 누르면 확인 상태가 되고, 제한 시간 안에 다시 눌러야 판매됩니다.
- 배속 버튼은 `1x / 2x / 3x` 순환으로 동작합니다.
- 감속 중인 적에게 냉각 링을 표시하고, 갑각 적에게 장갑 표시를 추가했습니다.

## 4차 작업 내용

- Web Audio 기반 효과음 시스템과 `소리/음소거` 버튼을 추가했습니다.
- 타워가 3단계로 강화될 때 최종 강화 분기를 선택할 수 있습니다.
- 펄스, 레이저, 플라즈마, 크라이오 타워에 각각 2개씩 분기 특성을 추가했습니다.
- 특수 배치 슬롯을 추가했습니다: 고지대, 냉각, 반응로, 광물.
- 특수 슬롯은 사거리, 공격 주기, 피해량, 처치 합금 보상에 실제 보너스를 줍니다.
- 플라즈마 산성 잔류장, 레이저 분광 빔, 크라이오 균열/정지, 펄스 관통/충격 효과를 추가했습니다.
- 기지 화면에 저장 데이터 초기화 버튼을 추가했습니다.

## 추가 보정

- 크라이오 타워 이름과 하단 구매 버튼 이름을 `크라이오 필드`로 통일했습니다.
- 빈 특수 슬롯을 터치하면 슬롯 보너스 설명이 선택 패널에 표시됩니다.
- 자동 웨이브 버튼을 추가했습니다.
- Easy, Normal, Hard, Hell 난이도 선택을 추가했고 난이도별 적 체력 배율을 적용했습니다.
- 실제 모바일 화면을 확인하며 경로와 겹치던 슬롯 좌표를 조정했습니다.

## 5차 작업 내용

- 신규 스테이지 `감염된 정제소`를 추가했습니다.
- 신규 스테이지 `심층 둥지 균열`을 추가했습니다.
- 스테이지 선택 화면이 6개 스테이지와 잠금 상태를 표시합니다.
- 스테이지별 특수 규칙을 추가했습니다.
- 특수 규칙은 웨이브 적 수, 적 체력, 보스 소환 주기에 실제 반영됩니다.
- 웨이브 생성은 JavaScript 로직으로 통일했고 스테이지 규칙 보정을 받습니다.
- 신규 스테이지를 포함한 전체 슬롯과 경로 간격을 재검증했습니다.

## 추가 확장

- 신규 스테이지 `궤도 추락지`를 추가했습니다.
- 신규 스테이지 `하이브 코어`를 추가했습니다.
- 전체 스테이지 수를 10개로 확장했습니다.
- 모든 스테이지 데이터는 `src/data/gameData.js`에서 순서대로 관리합니다.
- 난이도는 새 진행 데이터에서 최초 1회만 선택할 수 있고, 선택 후 저장 데이터 초기화 전까지 잠깁니다.
- 난이도 잠금 상태와 안내 문구를 기지 화면에 표시합니다.
- 난이도 선택은 기지 화면의 별도 팝업으로 표시되며, 각 난이도별 설명과 적 HP 배율을 함께 보여줍니다.
